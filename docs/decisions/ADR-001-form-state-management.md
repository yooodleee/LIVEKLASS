# ADR-001 폼 상태 관리 방식

## 결정 배경

다단계 폼(3 스텝)에서 스텝 간 이동 시 입력값을 유지해야 하며, 신청 유형(INDIVIDUAL/GROUP)에 따라 조건부 필드가 분기된다. 폼 전체 상태를 어떻게 관리할지 결정이 필요했다.

## 선택한 방식

React Hook Form의 `useForm`을 최상위 컴포넌트에서 단일 인스턴스로 선언하고, `FormProvider`와 `useFormContext()`로 각 스텝 컴포넌트에 공유한다.

## 고려한 대안

- **useState로 직접 관리**: 각 스텝의 입력값을 상위 컴포넌트 state로 관리하는 방식
- **Zustand/Jotai 전역 상태**: 별도 상태 라이브러리로 폼 데이터를 관리하는 방식
- **스텝별 독립 useForm**: 각 스텝마다 별도 useForm 인스턴스를 사용하고 제출 시 병합하는 방식

## 선택 이유

React Hook Form의 비제어 컴포넌트 방식은 입력 시 리렌더링을 최소화한다. `FormProvider` 패턴으로 단일 인스턴스를 공유하면 스텝 이동 시 별도 상태 직렬화/역직렬화 없이 입력값이 자동 유지된다.

## 트레이드오프

- **장점**: 리렌더링 최소화, 스텝 간 데이터 자동 유지, `trigger()`로 스텝별 부분 검증 가능
- **단점**: `FormProvider` 컨텍스트 범위 밖에서는 폼 상태에 접근 불가, 최상위 컴포넌트에 의존성 집중
